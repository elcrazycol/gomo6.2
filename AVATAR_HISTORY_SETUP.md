# Настройка истории аватаров

## Что добавлено

1. **Миграция базы данных** (`apps/backend-go/migrations/024_avatar_history.sql`):
   - Таблица `avatar_history` для хранения истории аватаров
   - Триггер для автоматического добавления аватаров в историю при обновлении профиля
   - RPC функции `get_avatar_history` и `delete_avatar_from_history`
   - Миграция существующих аватаров в историю

2. **Компонент AvatarGallery** (`apps/web/src/components/AvatarGallery.tsx`):
   - Просмотр истории аватаров как галереи
   - Навигация между аватарами (стрелки, свайпы, миниатюры)
   - Кнопка удаления для владельца профиля
   - Индикатор текущего аватара

3. **Обновлен Profile.tsx**:
   - Загрузка истории аватаров
   - Клик по аватару открывает галерею
   - Удаление аватаров с подтверждением

## Применение миграции

### Вариант 1: Через Supabase Dashboard
1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Скопируйте содержимое `apps/backend-go/migrations/024_avatar_history.sql`
4. Выполните SQL

### Вариант 2: Через Supabase CLI
```bash
supabase db push
```

### Вариант 3: Напрямую через psql
```bash
psql "postgresql://..." < apps/backend-go/migrations/024_avatar_history.sql
```

## Как это работает

1. **Загрузка нового аватара**: При загрузке нового аватара через профиль, триггер автоматически добавляет его в `avatar_history` и помечает как текущий.

2. **Просмотр истории**: Клик по аватару в профиле открывает галерею со всеми предыдущими аватарами. Можно листать стрелками, свайпами или кликать по миниатюрам.

3. **Удаление аватара**: 
   - Владелец профиля видит кнопку удаления (корзина) в галерее
   - При удалении текущего аватара, предыдущий становится активным
   - Удаление требует подтверждения

4. **Безопасность**: RPC функция `delete_avatar_from_history` проверяет, что пользователь удаляет только свои аватары.

## Особенности

- История аватаров сохраняется для всех пользователей
- Текущий аватар помечен в галерее
- Круглые миниатюры в галерее (как аватары)
- Поддержка мобильных устройств (свайпы, fullscreen)
- Автоматическая миграция существующих аватаров
