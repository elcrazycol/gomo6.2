# Messenger Audit & Improvements Plan

## Summary of findings

The messenger is well-built with proper encryption at rest (AES-256-GCM), membership checks on all endpoints, rate limiting, optimistic UI updates, and real-time WebSocket delivery. However, there are several security gaps, bugs, and UX issues to address.

---

## 1. Security Fixes

### 1a. WebSocket room subscription bypass (CRITICAL)
**File**: `apps/backend-go/internal/websocket/client.go:128-137`
- Any authenticated user can subscribe to `chat_{convID}` for conversations they don't belong to
- **Fix**: Validate room membership before subscribing. Add a DB check in `SubscribeToRoom` for chat rooms, or validate in `readPump` when handling `subscribe` messages for `chat_*` rooms.

### 1b. Rate limiter fragile type assertion
**File**: `apps/backend-go/internal/middleware/messenger_rate_limit.go:97`
- Uses `interface{ GetUserID() string }` duck-typing instead of direct `*auth.Claims` assertion
- **Fix**: Import `auth` package and assert directly to `*auth.Claims`

### 1c. Encryption key padding is insecure
**File**: `apps/backend-go/internal/api/handlers/messenger_encryption.go:28-31`
- Keys shorter than 32 bytes are zero-padded, which weakens the key
- **Fix**: Log a fatal error and refuse to start if key is wrong length, or use PBKDF2 to derive a proper key

### 1d. Missing input validation on UUIDs
**File**: `apps/backend-go/internal/api/handlers/messenger_messages.go` (and conversations, receipts)
- Path params like `conversationId` are passed directly to SQL without format validation
- **Fix**: Add UUID format validation helper, check at entry point of each handler

### 1e. XSS via `javascript:` URLs
**File**: `apps/web/src/components/messenger/MessageLinks.ts:76`
- External links are rendered with `href={url}`. While React auto-escapes, a `javascript:` scheme URL could still execute code
- **Fix**: Filter URLs to only allow `http:` and `https:` schemes in the external link renderer

## 2. Bug Fixes

### 2a. Stale typing indicator (never clears)
**File**: `apps/web/src/stores/messengerStore.ts:459-464`
- When a user types and disconnects without sending, "печатает..." stays forever
- **Fix**: Add a 5-second timeout in `setTyping` to auto-clear typing state. Store typing expiry timestamp and check in a periodic sweep.

### 2b. Optimistic edit not reverted on failure
**File**: `apps/web/src/stores/messengerStore.ts:318-329`
- `editMessage` updates UI immediately but doesn't revert if API fails
- **Fix**: Save original content before optimistic update, revert in catch block

### 2c. Conversation not moved to top on WS message
**File**: `apps/web/src/stores/messengerStore.ts:476-497`
- `updateConversationFromWs` updates fields but doesn't reorder conversations by `last_message_at`
- **Fix**: Sort conversations after updating (descending by `last_message_at`)

### 2d. New message count not reset on manual scroll
**File**: `apps/web/src/components/messenger/ChatView.tsx:79-97`
- When user scrolls down manually (not via button), `newMessageCount` keeps incrementing
- **Fix**: Reset `newMessageCount` to 0 when scroll reaches bottom in `handleScroll`

### 2e. Reply quote shows stale content if original deleted
**File**: `apps/web/src/components/messenger/MessageBubble.tsx:130-139`
- Quoted message content is captured at render time, not from live state
- **Fix**: Look up the quoted message from the store in ChatView and pass only the current content

### 2f. Dead code: `generateClientId()` in utils.ts
**File**: `apps/web/src/components/messenger/utils.ts:40-42`
- Exported but never imported (ChatView creates its own)
- **Fix**: Remove the unused function

## 3. UX / Mobile Improvements

### 3a. Safe area inset for chat scroll on iOS
**File**: `apps/web/src/index.css:1804`
- Message scroll area doesn't account for iPhone home indicator
- **Fix**: Add `padding-bottom: env(safe-area-inset-bottom)` to `.message-scroll` in the phone breakpoint

### 3b. Conversation list search/filter
**File**: `apps/web/src/components/messenger/ConversationList.tsx`
- No way to search through conversations when the list is long
- **Fix**: Add a search input at the top of the sidebar that filters conversations by username

### 3c. Keyboard shortcut: Escape to close chat on mobile
**File**: `apps/web/src/components/messenger/ChatView.tsx`
- No keyboard shortcut support for desktop users
- **Fix**: Add Escape key handler to go back to conversation list

### 3d. Double-tap to like/reply gesture on mobile
**File**: `apps/web/src/components/messenger/MessageBubble.tsx`
- Only long-press works on mobile, no double-tap
- **Fix**: Add double-tap handler for quick reply (optional, low priority)

---

## 4. Code Quality

### 4a. Type-safe WS message parsing
**File**: `apps/web/src/services/messengerWebSocket.ts:70-86`
- Uses `as string`, `as boolean` casts everywhere
- **Fix**: Add a `parseChatMessage(data: unknown): MessageView` function with runtime validation

### 4b. Remove unused imports
- Various files may have unused imports after changes

---

## Implementation Order

1. **Security first**: 1a (WS room bypass), 1e (XSS), 1c (key padding), 1d (UUID validation), 1b (rate limiter)
2. **Critical bugs**: 2a (typing stale), 2b (edit revert), 2c (conversation reorder), 2d (new msg count)
3. **UX improvements**: 3a (safe area), 3b (search), 3c (Escape key)
4. **Code quality**: 4a (type-safe WS), 2e (stale quote), 2f (dead code)
