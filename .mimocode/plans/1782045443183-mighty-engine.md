# Fix: Gift detail dialog shows wrong message

## Root Cause

`GiftDetailDialog` always receives `conversation.other_user_id` as `recipientId`. But the recipient is the non-sender, not always the other user.

Scenario: Bob sends Alice a gift. Alice opens chat → `other_user_id` = Bob. Dialog queries `user_gifts?recipient_id=eq.Bob` → fetches gifts Bob received (wrong!). Shows a completely different gift's message.

## Changes

### File: `apps/web/src/components/messenger/ChatView.tsx`

**1. Add state for gift detail recipient (near line 55):**

```tsx
const [giftDetailRecipientId, setGiftDetailRecipientId] = useState<string | null>(null);
```

**2. Fix click handler to determine correct recipient (line 329):**

Replace:
```tsx
onClick={() => setGiftDetailId(giftData.giftId)}
```

With:
```tsx
onClick={() => {
  setGiftDetailId(giftData.giftId);
  setGiftDetailRecipientId(
    msg.sender_user_id === me.id ? conversation.other_user_id : me.id
  );
}}
```

**3. Fix GiftDetailDialog call (line 413-418):**

Replace `recipientId={conversation.other_user_id}` with `recipientId={giftDetailRecipientId ?? me.id}`:

```tsx
{giftDetailId && (
  <GiftDetailDialog
    giftId={giftDetailId}
    recipientId={giftDetailRecipientId ?? me.id}
    open={true}
    onOpenChange={(v) => { if (!v) { setGiftDetailId(null); setGiftDetailRecipientId(null); } }}
  />
)}
```
