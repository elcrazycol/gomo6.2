# Исправление системы покупки Drops через DePay

## Диагноз

Деньги списываются с кошелька, но капли не зачисляются. Найдено 4 бага:

### Баг 1 (КРИТИЧЕСКИЙ): Нет `user_id` в payload
- **Frontend** (`DropsShop.tsx:61`) отправляет `payload: { drops_amount }` — **без `user_id`**
- **Backend** (`drops.go:238-240`) ждёт `payload.user_id` чтобы понять кому зачислить
- **Итог**: даже если коллбэк приходит, бэкенд не знает кому зачислить капли

### Баг 2 (КРИТИЧЕСКИЙ): Коллбэк не приходит на бэкенд
- В логах бэкенда **нет ни одного** вызова `DropsCallback` (нет `"[Drops] Callback received"`)
- Скорее всего URL коллбэка не настроен в DePay dashboard
- Нужно: `https://gomo6.wtf/api/v1/drops/callback`

### Баг 3: Подпись DePay не проверяется
- `drops.go:134-136` и `226-228` — проверка подписи закомментирована

### Баг 4: Config не валидирует user_id
- Config эндпоинт не проверяет что user_id принадлежит реальному пользователю

## План исправления (6 фаз)

### Фаза 1: Критические баги (исправление сломанного флоу)

#### 1.1 Frontend: Добавить `user_id` в payload
**Файл**: `apps/web/src/components/DropsShop.tsx`

Перед открытием DePay виджета — получить текущего юзера и добавить `user_id` в payload:

```typescript
// В handlePurchase, перед DePayWidgets.Payment:
const { data: { user } } = await api.auth.getUser();
if (!user) {
  toast.error("Нужно войти в аккаунт");
  setPurchasing(false);
  return;
}

DePayWidgets.Payment({
  integration: import.meta.env.VITE_DEPAY_INTEGRATION_ID || "",
  payload: {
    drops_amount: dropsAmount,
    user_id: user.id,  // ← КЛЮЧЕВОЕ ИСПРАВЛЕНИЕ
  },
});
```

#### 1.2 Backend: Валидация user_id в config
**Файл**: `apps/backend-go/internal/api/handlers/drops.go`

В `DropsConfig` после парсинга запроса — проверить что user_id существует:

```go
if req.UserID == "" {
    c.JSON(http.StatusBadRequest, models.ErrorResponse("user_id is required"))
    return
}
var userExists bool
err := h.db.QueryRow("SELECT EXISTS(SELECT 1 FROM users WHERE id = $1)", req.UserID).Scan(&userExists)
if err != nil || !userExists {
    c.JSON(http.StatusBadRequest, models.ErrorResponse("Invalid user"))
    return
}
```

#### 1.3 Backend: Включить проверку подписи DePay на коллбэке
**Файл**: `apps/backend-go/internal/api/handlers/drops.go`

Заменить закомментированный блок в `DropsCallback` на полную реализацию:
- Читаем raw body
- Декодируем `x-signature` (base64url)
- Верифицируем RSA-PSS (SHA256, salt=64)
- Если `DEPAY_PUBLIC_KEY` не задан — пропускаем с предупреждением в логах

Добавить импорты: `"io"`, `"bytes"`

### Фаза 2: Таблица pending-записей (трекинг)

#### 2.1 Новая миграция
**Файл**: `apps/backend-go/migrations/049_drops_pending.sql` (НОВЫЙ)

```sql
CREATE TABLE IF NOT EXISTS drops_pending (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    drops_amount INTEGER NOT NULL,
    price_usd DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'initiated'
        CHECK (status IN ('initiated', 'callback_received', 'credited', 'expired')),
    blockchain VARCHAR(50),
    tx_hash VARCHAR(255),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    confirmed_at TIMESTAMPTZ,
    credited_at TIMESTAMPTZ
);
CREATE INDEX idx_drops_pending_user ON drops_pending(user_id, status, created_at DESC);
CREATE INDEX idx_drops_pending_stale ON drops_pending(status, created_at) WHERE status = 'initiated';
```

#### 2.2 Config создаёт pending-запись
В `DropsConfig`, после расчёта цены, перед возвратом ответа:

```go
var pendingID string
_ = h.db.QueryRow(`
    INSERT INTO drops_pending (user_id, drops_amount, price_usd)
    VALUES ($1, $2, $3) RETURNING id
`, req.UserID, req.DropsAmount, priceUSD).Scan(&pendingID)
```

#### 2.3 Коллбэк отмечает pending как credited
В `DropsCallback`, после зачисления каплей:

```go
_, _ = tx.Exec(`
    UPDATE drops_pending
    SET status = 'credited', credited_at = NOW(), blockchain = $1, tx_hash = $2
    WHERE user_id = $3 AND drops_amount = $4 AND status = 'initiated'
      AND created_at > NOW() - INTERVAL '1 hour'
`, req.Blockchain, req.Transaction, payload.UserID, dropsAmount)
```

### Фаза 4: Эндпоинт ручной верификации (фоллбэк)

#### 4.1 Новый эндпоинт
**Файл**: `apps/backend-go/internal/api/handlers/drops.go`

`POST /api/v1/drops/manual-verify` (protected) — пользователь вводит tx_hash, бэкенд:
1. Проверяет что транзакция ещё не зачислена
2. Находит последний pending для этого юзера за 24 часа
3. Зачисляет капли и записывает транзакцию

#### 4.2 Регистрация маршрута
**Файл**: `apps/backend-go/internal/api/routes/routes.go`

```go
rest.POST("/drops/manual-verify", dropsHandler.ManualVerify)
```
