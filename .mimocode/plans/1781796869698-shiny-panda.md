# Messenger: Clickable Links + Rich Previews

## Problem

Message content renders as plain text (`<p>{message.content}</p>`). URLs are not clickable, internal links (gsub invites, threads, profiles) show no preview.

## Plan

### 1. New utility: `parseMessageLinks(content)` 

**File:** `apps/web/src/components/messenger/MessageLinks.tsx` (new)

Pure function that splits message text into segments: `text` | `link`. Detects URLs via regex.

**Regex ordering (critical — longest patterns first to avoid greedy match):**
1. Invite: `/g/{slug}/join/{code}` (longest, match first)
2. Thread with gsub: `/g/{slug}/thread/{id}`
3. Thread without: `/{slug}/thread/{id}`
4. Profile: `/profile/{userId}`
5. Board with gsub: `/g/{slug}`
6. Board without: `/{slug}`
7. External: `https?://...` (any valid HTTP URL — reject `javascript:`, `data:`, `file:`)

**XSS protection:**
- External URLs: strictly require `https?://` prefix — no other protocols
- External `<a>` tags: always `rel="noopener noreferrer" target="_blank"`
- Never use `dangerouslySetInnerHTML`

**Domain matching:** regex handles both production (`gomo6.wtf`) and local (`localhost:5173`, `localhost:8081`).

Returns: `Array<{ type: 'text', content: string } | { type: 'link', url: string, linkType: string, params: Record<string, string> }>`

### 2. New component: `MessageContent`

**File:** `apps/web/src/components/messenger/MessageContent.tsx` (new)

Replaces `<p>{message.content}</p>` in MessageBubble.

Renders parsed segments:
- `text` segments → plain text with `whitespace-pre-wrap`
- `link` segments → `<a>` tag, internal links use `<Link>` from react-router

For internal link types, render rich preview cards below the link text.

#### Loading/error handling (prevent layout shift)
- **Loading**: Render skeleton card `h-[64px] animate-pulse rounded-lg bg-muted/30` — fixed height prevents scroll jump
- **Error (404, network)**: Return `null` (hide preview, keep clickable link text) — don't spam error states in chat
- **Empty data**: Return `null`

#### Invite preview card
Fetch: `GET /api/v1/invites/{code}` (already exists, public)
Shows: board name, "Вас пригласили в {name}", `[Вступить →]` button linking to `/g/{slug}/join/{code}`
Mobile: full-width card, large touch target for button

#### Thread preview card  
Fetch: `GET /api/v1/threads?id=eq.{threadId}&select=id,title,post_count,boards(name,slug,is_gomosub)` (existing endpoint)
Shows: thread title (with `word-break: break-word`), post count, board name badge
Mobile: truncate title at 2 lines, keep compact

#### Profile preview card
Fetch from `ProfileCacheProvider.loadProfile(userId)` (already cached)
Shows: username with color, avatar, admin badge
Mobile: compact inline card

#### Board preview card
Fetch: `GET /api/v1/boards?slug=eq.{slug}&select=id,name,description,is_gomosub`
Shows: board name, description snippet (truncated)
Mobile: compact

#### External links
Just `<a href="..." target="_blank" rel="noopener noreferrer">` with underline styling

**File:** `apps/web/src/components/messenger/MessageBubble.tsx`

### 5. Debounce preview fetches

Each `MessageContent` component fetches its own preview data. With 50+ messages visible, this could be a lot of API calls.

Solution: Use React Query with `staleTime: 5min` + `gcTime: 10min`. Same URL = same cache key = 1 fetch total. `refetchOnMount: false`. This means scrolling through 50 messages with links only fetches each unique URL once.
