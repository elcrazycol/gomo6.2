# План: Подарок всегда отправляет сообщение в мессенджер + "Подарок" в превью

## Проблема

1. При отправке подарка **не из мессенджера** (со страницы профиля, `GiftsTab.tsx`) — сообщение `__GIFT__` **не создаётся** в мессенджере.
2. В списке чатов превью показывает сырой текст `__GIFT__:uuid:GiftName:imageUrl` вместо "🎁 Подарок".

## Решение

### Часть 1: Бэкенд — `gifts.go`

**Файл:** `apps/backend-go/internal/api/handlers/gifts.go`

В `SendGift` **после** `tx.Commit()` и `sendGiftNotification` — добавить блок мессенджера (ошибки логируются, не падают):

1. **Расширить запрос подарка** (строка 103): вместо `SELECT price` сделать `SELECT name, COALESCE(image_url, ''), price` — это уже в транзакции, данные нужны для контента сообщения.

2. **Найти/создать диалог** — retry loop (3 попытки):
   ```sql
   -- Поиск существующего 1:1 диалога
   SELECT cm1.conversation_id FROM chat_members cm1
   JOIN chat_members cm2 ON cm1.conversation_id = cm2.conversation_id
   WHERE cm1.user_id = $1 AND cm2.user_id = $2
     AND (SELECT COUNT(*) FROM chat_members WHERE conversation_id = cm1.conversation_id) = 2
   LIMIT 1
   ```
   Если `ErrNoRows` — создать:
   ```sql
   INSERT INTO chat_conversations DEFAULT VALUES RETURNING id
   INSERT INTO chat_members (conversation_id, user_id) VALUES ($1, $2), ($1, $3)
   ```

3. **Создать сообщение** — plaintext контент `__GIFT__:{giftID}:{giftName}:{giftImageURL}`, зашифровать через `encryptContent()`, вставить в `chat_messages`:
   ```sql
   INSERT INTO chat_messages (conversation_id, sender_user_id, content, client_id)
   VALUES ($1, $2, $3, $4) RETURNING id
   ```
   `client_id` = `"gift_{giftRecordID}"` (уникальный, для идемпотентности).

4. **Обновить `chat_conversations`**:
   ```sql
   UPDATE chat_conversations
   SET last_message_preview = '🎁 Подарок',
       last_message_sender_id = $1,
       last_message_id = $2,  -- ID созданного сообщения!
       updated_at = NOW()
   WHERE id = $3
   ```

5. **WS broadcast** — передавать **plaintext** контент (не зашифрованный):
   ```go
   hub.PublishNewChatMessage(gin.H{
       "id": msgID, "conversation_id": convID,
       "sender_user_id": senderID.String(),
       "content": giftContent,  // plaintext __GIFT__:...
       "client_id": clientID,
       ...
   })
   ```

6. **Инвалидировать кэш** мессенджера через `invalidateMessengerCaches`.

### Часть 2: `truncatePreview` — "🎁 Подарок" в превью

**Файл:** `apps/backend-go/internal/api/handlers/messenger.go`

```go
func truncatePreview(s string) string {
    if strings.HasPrefix(s, "__GIFT__") {
        return "🎁 Подарок"
    }
    runes := []rune(s)
    if len(runes) <= 80 { return s }
    return string(runes[:80])
}
```

`last_message_preview` хранится в plaintext → `truncatePreview` корректно распознаёт `__GIFT__`.

### Часть 3: Фронтенд — убрать дублирование

**Файл:** `apps/web/src/components/messenger/UserInfoPanel.tsx`

Убрать строки 87-91:
```typescript
// УБРАТЬ:
const giftContent = `__GIFT__:${selectedGift.id}:${selectedGift.name}:${selectedGift.image_url || ""}`;
const clientId = `g${Date.now().toString(36)}_${Math.random().toString(36).slice(2, 8)}`;
const sendMessage = useMessengerStore.getState().sendMessage;
await sendMessage(giftContent, clientId);
```

Остальная логика (toast, очистка формы, закрытие диалога) **не завязана** на `sendMessage` — триггером успешной отправки является ответ `/api/v1/gifts/send`.
