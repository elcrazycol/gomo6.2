# Plan: Test Coverage for client.ts and websocket.ts

## Context

`client.ts` (637 lines) — the core API client. `websocket.ts` (427 lines) — real-time WebSocket service. Both are critical infrastructure used by every feature. Current frontend coverage: ~18%. `client.auth.test.ts` exists but only covers auth/2FA/password methods. `websocket.ts` has 0% coverage.

## Strategy

Two test files, built incrementally. Each chunk tested and verified before moving to the next. One branch: `test/client-coverage`.

---

## File 1: `apps/web/src/integrations/api/client.test.ts`

Uses same patterns as `client.auth.test.ts`: `mockFetch` helpers, `vi.fn()` for global fetch, `beforeEach` cleanup.

### Chunk 1: Token management + constructor (10 tests)
- `setTokens` saves both tokens to localStorage
- `setTokens` with null refresh_token doesn't store refresh
- `clearTokens` clears everything + cachedUser
- `getToken`, `getRefreshToken` return correct values
- Constructor loads tokens from localStorage + decodes expiry
- Constructor with invalid JWT in localStorage — graceful

### Chunk 2: `tryRefreshToken()` (6 tests)
- No refresh token → returns null
- Successful refresh → updates tokens + returns new token
- Failed refresh (non-ok response) → returns null
- Network error → returns null
- Concurrent calls → deduplicates (single fetch)
- Keeps old refresh token if server doesn't send new one

### Chunk 3: `request()` core (8 tests)
- Adds Authorization header when token set
- No Authorization header when no token
- Non-JSON response → wraps as error
- HTTP error (non-401) → throws with status
- `success: false` in JSON body → throws
- Proactive refresh when token near expiry (<5 min)
- 401 retry with refresh → re-fetches with new token
- 401 no refresh token → clears tokens + dispatches `auth:expired` + throws

### Chunk 4: Board/Thread/Post methods (12 tests)
- `getBoards` with no params → GET `/api/v1/boards`
- `getBoards` with slug → query param `slug=eq:slug`
- `getBoards` with is_gomosub → query param `is_gomosub=eq:true`
- `getBoard(slug)` → GET `/api/v1/boards/{slug}`
- `createBoard` → POST with JSON body
- `getThreads` with board_id → query param
- `getThread(id)` → GET `/api/v1/threads/{id}`
- `createThread` → POST to `/api/rpc/create_thread`
- `getPosts` with thread_id → query param
- `getPost(id)` → GET `/api/v1/posts/{id}`
- `createPost` → POST to `/api/rpc/create_post`
- `getPosts` with limit/offset → pagination params

### Chunk 5: Profile + Like + RPC methods (10 tests)
- `getProfiles` with username filter
- `getProfile(id)` → GET
- `updateProfile(id, data)` → PUT
- `likeThread` → POST
- `unlikeThread` → DELETE
- `likePost` → POST
- `unlikePost` → DELETE
- `getThreadLikes` with params
- `getPostLikesCount` → GET
- `hasUserLikedPost` → GET with encoded params

### Chunk 6: Notification + Passkey methods (8 tests)
- `getNotifications` with pagination
- `markNotificationAsRead(id)` → PUT
- `markAllNotificationsAsRead` → PUT
- `getUnreadNotificationsCount` → GET
- `getMessengerUnreadCount` → GET
- `beginPasskeyRegistration` → POST
- `finishPasskeyLogin` → POST + sets tokens
- `listPasskeys` → GET, returns credentials array

### Chunk 7: `getDeviceId()` (3 tests)
- Returns existing device_id from localStorage
- Generates new device_id if none stored
- Generated ID starts with "device_"

## File 2: `apps/web/src/services/websocket.test.ts`

Uses same `MockWebSocket` pattern from `messengerWebSocket.test.ts`.

### Chunk 1: Connection lifecycle (8 tests)
- `connect` creates WebSocket when token exists
- `connect` does nothing when no token
- `connect` debounce — rapid calls only create one WebSocket
- `handleOpen` sets connected + sends auth message
- `handleOpen` resets reconnect counter
- `handleClose` on clean close (1000) — no reconnect
- `handleClose` on abnormal close — schedules reconnect
- `disconnect` cleans up everything

### Chunk 2: Reconnection (5 tests)
- Exponential backoff: 1s → 2s → 4s → ...
- Max reconnect attempts (10) — stops trying
- No reconnect when no token in localStorage
- Reconnect timeout cleared on `disconnect()`
- `auth:expired` event triggers disconnect

### Chunk 3: Room subscriptions (6 tests)
- `subscribe(room)` → sends subscribe message when connected
- `subscribe(room)` → queues if not connected
- `unsubscribe(room)` → sends unsubscribe + removes from set
- `subscribeToNotifications(userId)` → subscribes to `notifications_{userId}`
- `subscribeToFeed()` → subscribes to `feed`
- `subscribeToThread(id)` → subscribes to room
- `resubscribeRooms` after reconnect — re-sends all subscriptions

### Chunk 4: Message handling (5 tests)
- `on(type, handler)` → returns unsubscribe function
- `off(type, handler)` → removes handler
- Incoming message → dispatches to registered handlers
- Handler error → doesn't crash, logs to console
- Multiple handlers for same type → all called

### Chunk 5: Send + properties (5 tests)
- `sendTyping(room)` → sends typing message
- `sendTyping` when not connected → no-op
- `connected` getter → returns connection state
- `rooms` getter → returns subscribed rooms array
- Empty room string → subscribe/unsubscribe are no-ops

### Chunk 6: Ping + error handling (3 tests)
- Ping sent every 30s when connected
- Ping not sent when disconnected
- `handleError` → doesn't throw (logged silently)
