# Fix: Request Storm & White Screen Crash

## Problem Summary

After ~30 minutes the page goes white. Console shows hundreds of `NS_BINDING_ABORTED` XHR errors and `TypeError: _e.boards is undefined`. Root cause: **ThreadCard fires 4-5 individual API calls per thread on mount**, multiplied by 20+ threads = 80-100+ requests on load. BoardRouter.tsx adds N+1 polling. Over 30 minutes the browser tab OOMs.

## Changes

### 1. Backend: Batch likes endpoint `GET /api/rpc/get_thread_likes_batch`

**File:** `apps/backend-go/internal/api/handlers/rpc_likes.go`
- Add `GetThreadLikesBatch(c *gin.Context)` handler
- Accepts `thread_ids=uuid1,uuid2,...` and optional `user_uuid`
- Returns `[{thread_id, likes_count, is_liked}]` in a single query
- Query: `SELECT thread_id, COUNT(*) as likes_count FROM thread_likes WHERE thread_id IN (...) GROUP BY thread_id` + optional `EXISTS` subquery for `is_liked`

**File:** `apps/backend-go/internal/api/routes/routes.go`
- Register: `rpc.GET("/get_thread_likes_batch", rpcHandler.GetThreadLikesBatch)`

### 2. Frontend: Rewrite ThreadCard to use batch likes from parent

**File:** `apps/web/src/components/ThreadCard.tsx`
- Remove `loadLikesData` callback and its useEffect
- Remove `loadRecentPosts` callback and its useEffect (posts preview)
- Accept new optional props: `likesCount`, `userLiked`, `recentLikers`, `recentPost`
- Parent components will pre-fetch all likes data in bulk and pass down

### 3. Frontend: Add batch likes fetching to ThreadFeed

**File:** `apps/web/src/components/ThreadFeed.tsx`
- After loading threads, call `get_thread_likes_batch` once for all visible thread IDs
- Pass likes data down to each ThreadCard via props
- Remove per-ThreadCard likes loading

### 4. Frontend: Add batch likes fetching to Index.tsx subscriptions feed

**File:** `apps/web/src/pages/Index.tsx`
- After loading subscription threads, call `get_thread_likes_batch` once
- Pass likes data down to ThreadCard via props

### 5. Frontend: Add batch likes fetching to Profile.tsx

**File:** `apps/web/src/pages/Profile.tsx`
- Same pattern: bulk-fetch likes after loading threads, pass as props

### 6. Frontend: Add batch likes fetching to GomoSubs.tsx

**File:** `apps/web/src/pages/GomoSubs.tsx`
- Same pattern

### 7. Fix BoardRouter.tsx N+1 polling

**File:** `apps/web/src/pages/BoardRouter.tsx`
- Replace individual per-thread post fetches in `loadThreads()` with a single batch: `POST /api/v1/posts?thread_id=in.(...)&latest=true`
- This already works in `Board.tsx:300` — just copy the pattern

### 8. Add AbortController to ThreadFeed and BoardRouter polling

**File:** `apps/web/src/components/ThreadFeed.tsx`
- Add AbortController to cancel stale fetch when `loadThreads` is called again or component unmounts

**File:** `apps/web/src/pages/BoardRouter.tsx`
- Add AbortController to cancel stale `loadThreads` calls
- Clear polling interval on unmount

### 9. Remove LikesCacheContext dependency from LikeButton for initial load optimization

**File:** `apps/web/src/contexts/LikesCacheContext.tsx`
- Keep as-is (used by LikeButton in Board.tsx thread detail views)
- ThreadCard will no longer use it (batch fetch from parent replaces it)

## Request count comparison

**Before (20 threads, logged in):**
- ThreadCard mount: 20 × 4 = 80 requests
- BoardRouter 30s poll: ~20 post requests
- Over 30 min: 80 + 1200 = **~1280 requests**

**After:**
- ThreadCard mount: 1 batch likes + 0 individual = **1 request**
- BoardRouter 30s poll: 1 batch post request
- Over 30 min: 1 + 60 = **~61 requests**

**Reduction: ~95%**
