# Fix: Mass duplicate API calls during registration+login flow

## Problem Summary

Console log shows 80+ redundant API requests during a single page load. Key issues:

### 1. `UserBadge` makes independent API calls for every post author
`UserBadge.tsx:41-77` — Each instance fetches `user_achievements` + `profile_customization` independently. On a board with 15 threads, that's 30+ redundant calls. Also calls `AdminBadge` → `loadProfile` → 4 more queries.

**Fix**: `UserBadge` should use `ProfileCacheContext` instead of raw API calls. It already wraps `AdminBadge` and `ProfileHoverCard`, both of which access cache data independently.

### 2. `AppLayout` duplicates data that `ProfileCacheContext` already provides
`AppLayout.tsx:772-829` — Independently fetches `user_roles`, `profiles`, `user_achievements` on every `user?.id` change.

**Fix**: Use `useProfileCache()` in `AppLayout` for `isModerator`, `currentUserUsername`, `currentUserColor` instead of raw queries.

### 3. `ProcessedContent` and `MentionLink` fetch `user_achievements` independently
`ProcessedContent.tsx:106-116`, `MentionLink.tsx:63-72` — Both fetch `user_achievements` for author color on every render.

**Fix**: Use `ProfileCacheContext` or extract a shared `useUserColor(userId)` hook backed by a React Query cache.

### 4. `Index.tsx` duplicates data fetching
`Index.tsx:105-109` — Fetches `user_roles`, `profiles`, `user_achievements` independently, same as AppLayout.

**Fix**: Use `ProfileCacheContext` for user data. Only `user_terms_acceptance` needs a separate fetch.

### 5. `auth/me` called ~20 times
- `useAuth()` uses React Query (deduped by queryKey) — good
- But `api.auth.getSession()` is called directly by many pages (`Board.tsx`, `Index.tsx`, `Thread.tsx`, `BoardRouter.tsx`, etc.) — each call hits `auth/me`
- `onAuthStateChange` also calls `apiClient.getCurrentUser()` → `auth/me`

**Fix**: `api.auth.getSession()` should cache the result. Add a short-lived cache (30s) to `apiClient.getCurrentUser()` to prevent thundering herd.

### 6. Query param `+` signs in `user_achievements` select
The `++++++++++++` in the URL is just `URLSearchParams.toString()` encoding spaces from template literal indentation. **Not a bug**, just visual noise. Could clean up the template literal indentation but it's cosmetic.

### 7. auth/refresh returns 401 "MIDDLEWARE: Invalid token"
After network interruption, the refresh token becomes invalid. This is expected when the session expires. The cascade of `NetworkError` → failed refresh → session expired is correct behavior but the error handling could be cleaner.

### 8. WebSocket reconnections
Multiple `wss://gomo6.wtf/ws` connections open/close — each page navigation disconnects and reconnects. Minor but adds to the noise.

## Implementation Plan

### Step 1: Create `useUserColor(userId)` hook
**File**: `apps/web/src/hooks/useUserColor.ts`

A shared React Query hook that fetches `user_achievements` and returns the username color. Replaces the duplicated color-fetching logic in `UserBadge`, `ProcessedContent`, `MentionLink`, `AppLayout`, `Index`.

### Step 2: Refactor `UserBadge` to use cache
**File**: `apps/web/src/components/UserBadge.tsx`

- Remove independent `api.from("user_achievements")` call
- Remove independent `getProfileCustomization()` call
- Use `useProfileCache()` for customization data
- Use `useUserColor()` for color

### Step 3: Refactor `AppLayout` to use cache
**File**: `apps/web/src/components/AppLayout.tsx`

- Replace `api.from("user_roles")`, `api.from("profiles")`, `api.from("user_achievements")` with `useProfileCache()`

### Step 4: Refactor `Index.tsx` to use cache
**File**: `apps/web/src/pages/Index.tsx`

- Replace `api.from("user_roles")`, `api.from("profiles")`, `api.from("user_achievements")` with `useProfileCache()`
- Keep `user_terms_acceptance` as separate call (not cached elsewhere)

### Step 5: Refactor `ProcessedContent` and `MentionLink`
**Files**: `apps/web/src/components/ProcessedContent.tsx`, `apps/web/src/components/MentionLink.tsx`

- Replace independent `user_achievements` calls with `useUserColor()`

### Step 6: Cache `getCurrentUser()` results briefly
**File**: `apps/web/src/integrations/api/client.ts`

Add a 30-second in-memory cache for `getCurrentUser()` to prevent thundering herd when multiple pages/components call it simultaneously. The `cachedUser` field already exists but is only used for network error fallback.
