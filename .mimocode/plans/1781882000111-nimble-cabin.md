# Fix iOS Flash + Safari Jitter v2

## Problems
1. **Flash**: CSS `height: 100dvh` is correct, but there's a brief flash when JS overrides it. Need to hide the panel until JS sets the correct height.
2. **Safari jitter**: Touch events provoke selection/scroll bounce on the fixed panel.

## Solution

### Fix 1: Hide panel until JS initializes height
- CSS: Add `opacity: 0` to `.chat-panel.is-open` on mobile (only when keyboard hasn't been opened yet)
- JS: In `updateHeight`, after setting height+transform, set `panel.style.opacity = "1"`
- This eliminates the flash — panel is invisible until JS has the correct dimensions

### Fix 2: Touch-action on panel, selective user-select
- CSS: `.chat-panel.is-open` gets `touch-action: manipulation` (prevents double-tap zoom, reduces jitter)
- CSS: `.chat-panel.is-open` gets `-webkit-user-select: none` on the container
- CSS: `.chat-panel.is-open .message-content p` gets `user-select: text` (restore text selection in messages)

## Files to Modify
