# Fix: isLocked race condition — banner shows for friends

## Problem
`isMutualFriend` defaults to `false`. The async check at line 397-401 sets it to `true` only after the API responds. Until then, `isLocked = true` for ALL non-owners, including mutual friends. The banner "Приватный профиль" + "Добавить в друзья" shows even for friends.

## Fix
One line change in Profile.tsx line 912:
```
// Before:
const isLocked = privateProfile && !isOwnProfile && !isMutualFriend;

// After:
const isLocked = privacyChecked && privateProfile && !isOwnProfile && !isMutualFriend;
```

`privacyChecked` is already set to `true` after the friendship check completes (line 404). Until then, `isLocked` is `false` — full profile renders. Once checked, if not a friend → locked; if friend → unlocked.
