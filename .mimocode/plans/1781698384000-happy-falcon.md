# Blockchain Nicknames System — Plan

## Vision

Unique blockchain nicknames on Base L2. Each nickname is an NFT (ERC-721) that can be transferred, sold, collected. Multiple nicknames per profile. Passkey-controlled wallet — no seed phrases, no gas fees for users.

## Key Design Decisions

### 1. Account Abstraction (ERC-4337)

User's passkey controls a **Smart Contract Wallet** on Base:
- Passkey signs `UserOperation` (via WebAuthn signature validation in the contract)
- Backend relayer submits `UserOperation` to Base EntryPoint
- User never handles ETH, gas, or private keys
- Even if backend disappears, wallet is on-chain and recoverable

### 2. NicknameRegistry = ERC-721

Each nickname is an NFT token:
- `mint(name, address)` — register new nickname
- `transfer(from, to, tokenId)` — sell/gift
- `balanceOf(owner)` — count nicknames
- `tokenOfOwnerByIndex(owner, index)` — enumerate collection
- `tokenURI(tokenId)` — metadata (could include image, rarity, etc.)

### 3. Registration Flow

```
1. User opens /auth → "Create Account"
2. Enter username + email + "Create Passkey" (WebAuthn)
3. Passkey created ✓
4. "Choose your blockchain nickname"
5. User types desired name → check availability (DB + on-chain)
6. If available → sign registration message with passkey
7. Backend:
   a. Deploy GomoWallet for user (if first time)
   b. Call NicknameRegistry.mint(name, walletAddress)
   c. Store in DB
8. Done! User sees "Your nickname @xxx is verified on Base"
```

### NicknameRegistry.sol (ERC-721)

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";

contract NicknameRegistry is ERC721 {
    mapping(uint256 => string) private _tokenNames;
    mapping(string => uint256) private _nameToTokenId;
    mapping(address => uint256) private _nonce; // for token ID generation

    event NicknameMinted(address indexed owner, string name, uint256 tokenId);
    event NicknameTransferred(address indexed from, address indexed to, string name);

    modifier onlyEntryPointOrSelf() {
        require(
            msg.sender == address(entryPoint()) || msg.sender == address(this),
            "Only EntryPoint"
        );
        _;
    }

    function mint(string calldata name, address to) external onlyEntryPointOrSelf {
        require(bytes(name).length >= 3 && bytes(name).length <= 32, "Invalid length");
        require(_nameToTokenId[name] == 0, "Name taken");
        require(_isAlphanumeric(name), "Invalid characters");

        _nonce[to]++;
        uint256 tokenId = uint256(keccak256(abi.encodePacked(to, _nonce[to], block.timestamp)));

        _mint(to, tokenId);
        _tokenNames[tokenId] = name;
        _nameToTokenId[name] = tokenId;

        emit NicknameMinted(to, name, tokenId);
    }

    function nameOf(uint256 tokenId) external view returns (string memory) {
        return _tokenNames[tokenId];
    }

    function tokenByName(string calldata name) external view returns (uint256) {
        return _nameToTokenId[name];
    }

    function isAvailable(string calldata name) external view returns (bool) {
        return _nameToTokenId[name] == 0
            && bytes(name).length >= 3
            && bytes(name).length <= 32
            && _isAlphanumeric(name);
    }

    function _isAlphanumeric(string calldata s) internal pure returns (bool) {
        bytes memory b = bytes(s);
        for (uint256 i = 0; i < b.length; i++) {
            if (
                (b[i] < 0x30 || b[i] > 0x39) && // 0-9
                (b[i] < 0x61 || b[i] > 0x7A) && // a-z
                (b[i] < 0x41 || b[i] > 0x5A) && // A-Z
                b[i] != 0x2D && b[i] != 0x5F     // - _
            ) return false;
        }
        return true;
    }

    // Allow transfer (default ERC-721)
    function transferFrom(address from, address to, uint256 tokenId) public virtual override {
        super.transferFrom(from, to, tokenId);
        emit NicknameTransferred(from, to, _tokenNames[tokenId]);
    }

    // Batch query
    function getNicknames(address owner) external view returns (string[] memory) {
        uint256 count = balanceOf(owner);
        string[] memory names = new string[](count);
        for (uint256 i = 0; i < count; i++) {
            uint256 tokenId = tokenOfOwnerByIndex(owner, i);
            names[i] = _tokenNames[tokenId];
        }
        return names;
    }
}
```

```sql
-- 047_blockchain_nicknames.sql

CREATE TABLE IF NOT EXISTS user_wallets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    wallet_address VARCHAR(42) NOT NULL UNIQUE,

    webauthn_credential_id BYTEA NOT NULL,

CREATE TABLE IF NOT EXISTS user_nicknames (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    nickname VARCHAR(32) NOT NULL UNIQUE,
    token_id VARCHAR(78) NOT NULL UNIQUE,  -- uint256 as string
    contract_address VARCHAR(42) NOT NULL,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    tx_hash VARCHAR(66) NOT NULL,
```

## Gas Payment Strategy

Two options:

### Option A: Master Wallet (Simpler, Recommended for MVP)
- Backend has a funded wallet (deployer key)
- All user transactions relayed through master wallet
- Master wallet pays gas on Base (~$0.001-0.01 per tx)
- Budget: ~$10-50/month for 1000+ users

### Option B: User-Funded (More Decentralized)
- Each user gets a tiny ETH amount from faucet on registration
- User pays their own gas
- More complex UX

**Recommendation**: Start with Option A. Move to Option B later when volume justifies it.

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Base RPC rate limits | High | Use paid RPC (Alchemy/Infura) or multiple providers |
| Contract bugs | High | Extensive testing on Sepolia before mainnet |
| Passkey compatibility | Medium | Graceful fallback, clear error messages |
| Gas costs spike | Low | Base is very cheap, master wallet budget |
| User confusion | Medium | Simple UX, skip blockchain details in UI |
