# Fix Friend System & Content Privacy

## Problems Found

### Critical: `privateHideXxx` settings are completely broken
`Profile.tsx:939` wraps ALL profile content in `{!isLocked && ...}`. Inside that block, every `(!isLocked || !privateHideXxx)` check (lines 943, 1171, 1183, 1195, 1207, 1222) is **always true** — we're already inside `!isLocked`. The granular privacy settings have zero effect. Non-friends either see everything or nothing.

### Critical: Friends see nothing when profile is private
`isLocked` = `privateProfile && !isOwnProfile && !isMutualFriend`. If `isMutualFriend` fails to set (fetch error, timing), friends see the locked profile with zero content. Also causes flicker: initially `isMutualFriend=false` → shows locked → then loads → shows full profile.

### Backend: Privacy filtering is incomplete
- `ShouldFilterPrivateProfile` in `profiles.go:174-181` only strips `avatar_url`, `bio`, `bio_json` — NOT `garma`, `post_count`, `thread_count`, `email`, `is_online`, `last_seen_at`.
- Threads listed by `user_id` on private profiles are visible to everyone (no friend check in `threads.go`).
- Gifts, achievements, wall posts go through UniversalHandler with no privacy check at all.
- Bio has no `private_hide_bio` setting — it's always stripped. This is fine, just needs to be consistent.

## Plan

### 1. Create shared backend privacy middleware

**File: `apps/backend-go/internal/api/handlers/privacy.go`**

Rename `ShouldFilterPrivateProfile` → keep it, but add a new convenience function:

```go
// CanViewUserContent returns true if the viewer can see the target user's content.
// Returns true if: profile is not private, viewer is owner, or viewer is mutual friend.
func CanViewUserContent(db *sql.DB, viewerID, targetUserID string) (bool, error) {
    shouldFilter, _, err := ShouldFilterPrivateProfile(db, viewerID, targetUserID)
    if err != nil {
        return false, err
    }
    return !shouldFilter, nil
}
```

Use this in all handlers that serve user-specific content.

### 2. Fix backend profile field stripping

**File: `apps/backend-go/internal/api/handlers/profiles.go`**

In `GetProfile` (line 239) and `GetProfiles` (line 174), when `shouldFilter` is true, also strip:
- `Garma` → 0
- `PostCount` → 0
- `ThreadCount` → 0
- `Email` → ""
- `IsOnline` → false
- `LastSeen` → nil

### 3. Add backend privacy check for user threads

**File: `apps/backend-go/internal/api/handlers/threads.go`**

In the thread listing handler (~line 262), when `user_id` filter is present:
1. Look up the thread owner from the query
2. Call `CanViewUserContent(db, viewerID, ownerID)`
3. If false, return empty results

This should be a reusable check, not inline logic.

### 4. Add backend privacy check for gifts

**File: `apps/backend-go/internal/api/handlers/gifts.go`**
