# План: Исправление проблем с клавиатурой в мессенджере (mobile)

## Проблемы
1. В Safari можно скроллить контент с открытой клавиатурой — прыжки, джанк
2. На iPhone в браузере под инпутом появляется пузырь с адресом сайта (Safari Smart Search Field)
3. При открытии/закрытии клавиатуры layout прыгает, контент прячется за клавиатурой
4. Composer textarea не растягивается по содержимому (multi-line)
5. На мобильных в целом experience нестабильный

## Корневые причины

1. **Нет `interactive-widget` в viewport meta** — браузер не знает как обрабатывать виртуальную клавиатуру
2. **Fragile visualViewport хаки** — напрямую ставится `height` и `transform` на `.chat-panel`, `opacity: 0` как crutch
3. **body overflow:hidden конфликтует с iOS keyboard** — блокирует нативное поведение ресайза viewport
4. **Нет auto-resize у textarea** — rows=1, max-height: 140px, но JS не вычисляет высоту
5. **scroll-behavior: smooth на .message-scroll** — джанк при анимации клавиатуры
6. **`user-select: none`** — блокирует копирование текста сообщений
7. **Safari Smart Search Field** — пузырь с URL появляется когда body скроллится

## Стратегия

Используем `interactive-widget=resizes-visual` — браузер сжимает **визуальный** viewport при открытии клавиатуры. Это значит:
- CSS Grid `grid-template-rows: auto 1fr auto` на `.chat-panel` **автоматически** поднимет composer над клавиатурой
- `visualViewport.height` уменьшается — можно использовать для точного позиционирования
- Не нужно вручную двигать `.chat-panel` через transform/height — CSS делает это сам
- При `resizes-visual` body НЕ скроллится → пузырь URL не появляется (в связке с `position: fixed` на контейнере)

## Шаги реализации

### Шаг 1: Viewport meta (`apps/web/index.html`)

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, interactive-widget=resizes-visual" />
```

### Шаг 2: CSS — зафиксировать контейнер вместо body overflow:hidden (`index.css`)

Заменить JS-based body overflow:hidden на чистый CSS. Добавить на `html, body`:
```css
html, body {
  overscroll-behavior: contain;
  height: 100%;
  overflow: hidden;
}
```

На `.messenger-app` (desktop):
```css
.messenger-app {
  height: calc(100dvh - 5rem);
  min-height: min(640px, calc(100dvh - 5rem));
  display: flex;
  flex-direction: column;
  overflow: hidden;
  border: 1px solid hsl(var(--border));
  border-radius: var(--radius);
  background: hsl(var(--background));
}
```

На `.messenger-app` (mobile ≤980px) — переписать:
```css
.messenger-app {
  position: fixed;
  inset: 0;
  height: 100dvh;
  height: 100svh; /* fallback для старых Safari */
  border: 0;
  border-radius: 0;
}
```

`position: fixed` + `inset: 0` изолирует контент от body scroll → **пузырь Safari не появляется**.

### Шаг 3: Переписать mobile CSS для chat-panel (`index.css`)

**Убрать** из `.chat-panel.is-open`:
- `opacity: 0` (было костылем для visualViewport JS)
- `touch-action: manipulation` (мешает скроллу сообщений)
- `-webkit-user-select: none; user-select: none` (блокирует копирование)

**Текущий CSS** (строки 1740-1752):
```css
.chat-panel.is-open {
  display: grid;
  position: fixed;
  inset: 0;
  z-index: 30;
  height: 100dvh;
  background: hsl(var(--background));
  grid-template-rows: auto 1fr auto;
  touch-action: manipulation;
  -webkit-user-select: none;
  user-select: none;
  opacity: 0;
}
```

**Новый CSS**:
```css
.chat-panel.is-open {
  display: grid;
  position: fixed;
  inset: 0;
  z-index: 30;
  background: hsl(var(--background));
  grid-template-rows: auto 1fr auto;
}
```

Убрали `height: 100dvh` — `position: fixed; inset: 0` и так даёт полный viewport. При `resizes-visual` браузер сожмёт визуальный viewport, а `inset: 0` послушно сожмёт `.chat-panel` вместе с ним. Явный `height: 100dvh` мог бы конфликтовать.

С `resizes-visual` + `grid-template-rows: auto 1fr auto`:
- `auto` = chat-topbar (всегда виден)
- `1fr` = message-scroll (сжимается когда клавиатура открыта)
- `auto` = composer (всегда виден, поднимается над клавиатурой)

**Ничего额外ного JS не нужно** — CSS Grid делает всё сам.

### Шаг 4: Удалить body overflow:hidden effect (`MessengerView.tsx`)

**Удалить** effect на строках 65-79:
```tsx
// ── Prevent body scroll (robust for iOS/Android) ──
useEffect(() => {
  const html = document.documentElement;
  const body = document.body;
  const prevHtmlOverflow = html.style.overflow;
  const prevBodyOverflow = body.style.overflow;
  html.style.overflow = "hidden";
  body.style.overflow = "hidden";
  return () => {
    html.style.overflow = prevHtmlOverflow;
    body.style.overflow = prevBodyOverflow;
  };
}, []);
```

Вместо этого — стили на html/body в CSS (Шаг 2).

### Шаг 5: Упростить / переписать visualViewport effect (`MessengerView.tsx`)

**Удалить** текущий iOS keyboard effect (строки 107-136), который напрямую мутит `.chat-panel`:
```tsx
// ── iOS keyboard: dynamic viewport height (mobile only) ──
useEffect(() => {
  if (!isMobile) return;
  const vv = window.visualViewport;
  if (!vv) return;
  const updateHeight = () => {
    const panel = document.querySelector(".chat-panel.is-open") as HTMLElement | null;
    if (panel) {
      panel.style.height = `${vv.height}px`;
      panel.style.transform = `translateY(${vv.offsetTop}px)`;
      panel.style.opacity = "1";
    }
  };
  vv.addEventListener("resize", updateHeight);
  vv.addEventListener("scroll", updateHeight);
  updateHeight();
  return () => { ... };
}, [isMobile, showMobileChat]);
```

С `resizes-visual` этот хак **не нужен** — CSS Grid сам адаптирует высоты. Удаляем целиком.

**Замечание**: если на каких-то старых устройствах `interactive-widget` не поддерживается, то CSS Grid `auto 1fr auto` всё равно корректно разделит пространство — просто body высота не изменится и клавиатура перекроет нижнюю часть. Это acceptable fallback.

### Шаг 6: Auto-resize textarea

**Приоритет: CSS-first подход** — используем `field-sizing: content` (поддерживается в Chrome 123+, Safari 17.4+, Firefox 131+):

```css
.composer textarea {
  field-sizing: content;
  min-height: 42px;
  max-height: 140px;
  overflow-y: auto;
  resize: none;
}
```

Это даёт нативный auto-resize без Forced Reflow. Для старых браузеров без `field-sizing` — JS-фолбэк **внутри handleChange** (не useEffect, чтобы избежать лишнего цикла рендеринга):

```tsx
const handleChange = useCallback((e: React.ChangeEvent<HTMLTextAreaElement>) => {
  const value = e.target.value;
  if (value.length <= MAX_LENGTH) setDraft(value);

  // JS fallback for auto-resize if field-sizing: content not supported
  if (!CSS.supports('field-sizing', 'content')) {
    const ta = e.target;
    ta.style.height = 'auto';
    ta.style.height = `${Math.min(ta.scrollHeight, 140)}px`;
  }

  // typing indicator...
}, [setDraft, onTyping]);
```

При отправке (`setDraft("")`) высота сбрасывается через CSS `min-height: 42px`.

.message-scroll {

### Шаг 8: Упростить visualViewport listener в ChatView.tsx

Оставить текущий listener (строки 91-102) — он делает `pinToBottom()` при `visualViewport.resize`. Это правильно: при открытии клавиатуры чат должен доскроллиться вниз. Никаких изменений не требуется — он уже работает корректно.

### Шаг 9: Фокус-менеджмент

Убедиться что при `handleSelectConversation` НЕ вызывается `composerRef.current?.focus()`. Клавиатура должна открываться только по тапу пользователя.

В `handleReply` (ChatView.tsx строка 130):
```tsx
setTimeout(() => composerRef.current?.focus(), 50);
```

Это OK — пользователь нажал "ответить", ожидает что клавиатура откроется. Оставляем как есть.
