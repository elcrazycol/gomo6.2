## Как протестировать

### 1. Запустите сервисы

```bash
# Backend (если еще не запущен)
cd apps/backend-go
docker-compose up -d

# Frontend
cd apps/web
npm run dev
```

### 2. Откройте два браузера

- **Браузер 1**: Chrome (обычный режим)
- **Браузер 2**: Chrome (режим инкогнито) или Firefox

### 3. Зарегистрируйте/войдите под разными пользователями

В каждом браузере:
1. Перейдите на `http://localhost:5173`
2. Войдите под разными аккаунтами (или создайте новые)

### 4. Начните диалог

**В Браузере 1:**
1. Перейдите в профиль второго пользователя
2. Нажмите кнопку "Написать сообщение" (или перейдите на `/messages?user={user_id}`)
3. Напишите первое сообщение

**В Браузере 2:**
1. Перейдите на `/messages`
2. Вы должны увидеть новый диалог в списке
3. Откройте диалог
4. Вы должны увидеть сообщение от первого пользователя

### 5. Проверьте работу в реальном времени

**Тест 1: Мгновенная доставка**
- Отправьте сообщение из Браузера 1
- Сообщение должно появиться в Браузере 2 **мгновенно** (без перезагрузки)

**Тест 2: Статусы доставки**
- Отправьте сообщение из Браузера 1
- Вы должны увидеть статус ">" (отправлено)
- Когда Браузер 2 откроет диалог, статус изменится на ">>" (доставлено)
- Когда Браузер 2 прочитает сообщение, статус станет зеленым ">>" (прочитано)

**Тест 3: Счетчик непрочитанных**
- Отправьте несколько сообщений из Браузера 1
- В Браузере 2 в списке диалогов должен появиться счетчик непрочитанных
- Откройте диалог в Браузере 2
- Счетчик должен обнулиться

**Тест 4: Шифрование**
- Отправьте сообщение с текстом "Секретное сообщение"
- Откройте DevTools → Application → Local Storage
- Найдите ключ `gomo6-messenger-sodium-state`
- Проверьте, что там есть `publicKey` и `privateKey`
- Откройте базу данных и проверьте таблицу `chat_messages`
- Поле `ciphertext` должно содержать нечитаемый зашифрованный текст

### 6. Проверьте WebSocket соединение

**В DevTools (F12):**
1. Перейдите на вкладку **Network**
2. Выберите фильтр **WS** (WebSocket)
3. Вы должны увидеть соединение с `ws://localhost:8080/ws?token=...`
4. Статус должен быть **101 Switching Protocols** (зеленый)
5. В Messages вы должны видеть:
   - `{"type":"connected",...}` при подключении
   - `{"type":"confirmation","data":{"action":"subscribe","room":"chat_..."}}` при открытии диалога
   - `{"type":"new_chat_message",...}` при получении нового сообщения

**В Console:**
```javascript
// Проверьте, что WebSocket подключен
console.log('[WebSocket] Connected')
```

### 7. Проверьте логи бэкенда

```bash
cd apps/backend-go
docker-compose logs -f backend | grep WebSocket
```

Вы должны увидеть:
```
[WebSocket] Client connected: username (user_id)
[WebSocket] Client username subscribed to room chat_conversation_id
[WebSocket DEBUG] Publishing new chat message event for conversation conversation_id
[WebSocket] Published chat message event for conversation conversation_id
[WebSocket] Broadcasted to room chat_conversation_id (2 clients)
```

## Возможные проблемы

### WebSocket не подключается

**Симптом:** В DevTools → Network → WS нет соединения

**Решение:**
1. Проверьте, что бэкенд запущен: `curl http://localhost:8080/health`
2. Проверьте переменную окружения: `VITE_WS_URL=ws://localhost:8080/ws`
3. Проверьте, что токен валидный (войдите заново)

### Сообщения не приходят в реальном времени

**Симптом:** Нужно перезагружать страницу, чтобы увидеть новые сообщения

**Решение:**
1. Проверьте WebSocket соединение (см. выше)
2. Проверьте, что Redis запущен: `docker-compose ps redis`
3. Проверьте логи: `docker-compose logs backend | grep "new_chat_message"`

### Ошибка "У собеседника пока нет messenger-ключа"

**Симптом:** Не можете отправить сообщение

**Решение:**
1. Второй пользователь должен хотя бы раз зайти на страницу `/messages`
2. Это создаст его публичный ключ в таблице `chat_user_keys`
3. После этого можно отправлять сообщения

### Ошибка "[Не удалось расшифровать сообщение]"

**Симптом:** Вместо текста сообщения показывается ошибка

**Решение:**
1. Очистите localStorage: `localStorage.clear()` в DevTools Console
2. Перезагрузите страницу
3. Войдите заново
4. Это создаст новую пару ключей
