# Исправления мессенджера - финальная версия

## Что было исправлено

### 1. RPC функции теперь вызываются через fetch API
Проблема была в том, что `supabase.rpc()` не работал с нашим бэкендом. Теперь все RPC вызовы идут напрямую через fetch:

- `get_or_create_direct_chat` - создание/получение диалога
- `chat_mark_delivered` - отметка о доставке
- `chat_mark_read` - отметка о прочтении

### 2. Добавлены WebSocket уведомления
- Автоматическое обновление сообщений в реальном времени
- Подписка на комнаты `chat_{conversation_id}`
- Публикация событий через Redis

### 3. Создан WebSocketContext
- Автоматическое переподключение
- Управление подписками
- Обработка событий

## Как протестировать

1. **Откройте два браузера** (Chrome обычный + Chrome инкогнито)

2. **Войдите под разными пользователями** в каждом браузере

3. **Начните диалог:**
   - В первом браузере перейдите в профиль второго пользователя
   - Нажмите "Написать сообщение" или перейдите на `/messages?user={user_id}`
   - Теперь должен создаться диалог (без бесконечной загрузки!)

4. **Отправьте сообщение:**
   - Напишите текст и отправьте
   - Во втором браузере откройте `/messages`
   - Сообщение должно появиться **мгновенно** через WebSocket

5. **Проверьте статусы:**
   - ">" - отправлено
   - ">>" серый - доставлено
   - ">>" зеленый - прочитано

## Технические детали

### Backend (Go)
- RPC handlers в `rpc.go` (строки 722-1000)
- WebSocket события в `hub.go` и `universal.go`
- Redis pub/sub для масштабирования

### Frontend (React)
- Прямые fetch вызовы вместо `supabase.rpc()`
- WebSocketContext для управления соединением
- Автоматическая подписка на комнаты чата

### API Endpoints
```
POST /rpc/v1/get_or_create_direct_chat
POST /rpc/v1/chat_mark_delivered
POST /rpc/v1/chat_mark_read
```

Все запросы требуют Bearer токен в заголовке Authorization.

## Если что-то не работает

1. **Проверьте бэкенд:** `curl http://localhost:8080/health`
2. **Проверьте логи:** `docker-compose logs backend | grep WebSocket`
3. **Проверьте консоль браузера:** DevTools → Console
4. **Проверьте Network:** DevTools → Network → WS (должно быть WebSocket соединение)
