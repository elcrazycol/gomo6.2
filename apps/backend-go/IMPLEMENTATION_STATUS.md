## 🧠 Архитектурные решения

### ✅ Выбранные подходы
1. **Custom федеративный протокол** - проще чем ActivityPub
2. **Supabase совместимость** - плавная миграция
3. **JWT аутентификация** - стандартно и надежно
4. **PostgreSQL** - мощная реляционная БД
5. **Gin фреймворк** - производительный HTTP

### 🔄 Федеративная модель
- **Пользователи**: `username@domain` (глобально уникальные)
- **G-сабы**: глобальные сообщества с владельцем
- **Треды**: хранятся на сервере создания
- **Межсерверное общение**: REST + WebSocket

## 🧪 Тестирование API

```bash
# Регистрация пользователя
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username": "testuser", "email": "test@example.com", "password": "password123"}'

# Создание доски
curl -X POST http://localhost:8080/rest/v1/boards \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"slug": "test-board", "name": "Test Board", "description": "A test board"}'

# Создание треда
curl -X POST http://localhost:8080/rest/v1/threads \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"board_id": "BOARD_ID", "title": "Test Thread", "content": "Test content"}'

# Создание поста
curl -X POST http://localhost:8080/rest/v1/posts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"thread_id": "THREAD_ID", "content": "Test post"}'
```
